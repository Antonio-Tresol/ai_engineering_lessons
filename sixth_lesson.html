Retrieval-Augmented Generation (RAG)

A technique that enhances a foundation model's output by first retrieving relevant information from an external data source and then providing that information as context for the generation process.

(Huyen, 2025, p. 253)

Why is RAG important?

It's crucial for mitigating model hallucinations, which often occur when a model lacks the necessary context to answer a query accurately. RAG remains critical even as model context windows expand, because application data needs often grow faster than context limits.

(Lewis et al., 2020; Huyen, 2025, p. 255)

RAG Architecture

A standard RAG system consists of two primary components:

  • 1. Retriever: Fetches information from an external source (e.g., a document database). Its functions include Indexing (structuring data) and Querying (finding relevant chunks).
  • 2. Generator: An LLM that uses the original prompt plus the retrieved context to produce the final response.

(Huyen, 2025, p. 256)

Retrieval Algorithms

Term-Based (Sparse) Retrieval

Matches documents based on keywords (e.g., TF-IDF, Okapi BM25).

Pros: Fast, inexpensive. Cons: Lacks semantic understanding.

(Robertson & Zaragoza, 2009, as cited in Huyen, 2025, p. 260)

Embedding-Based (Dense) Retrieval

Ranks documents based on meaning similarity using vector embeddings.

Pros: Captures semantic nuance. Cons: More expensive and slower.

(Huyen, 2025, p. 262)

Hybrid Search combines both methods to balance speed, cost, and accuracy, often using a term-based search followed by an embedding-based reranker.

(Cormack et al., 2009, as cited in Huyen, 2025, p. 267)

Retrieval Optimization & Scope

  • Chunking Strategy: How documents are split impacts quality. Overlapping chunks is recommended.
  • Query Rewriting: Using an LLM to reformulate ambiguous queries.
  • Contextual Retrieval: Augmenting chunks with metadata (title, summary) to make them more discoverable.
  • Beyond Text: RAG can be extended to Multimodal data (images via CLIP) and Tabular data (via Text-to-SQL workflows).

(Huyen, 2025, pp. 268-274)

AI Agents

An agent is an entity that can perceive its environment and act upon it. In AI engineering, an agent uses an LLM as its "brain" to process information, create a plan, and execute actions using a set of available tools.

(Russell & Norvig, 1995, as cited in Huyen, 2025, p. 275)

Agent Planning & Execution

  • Decoupling Planning and Execution: Best practice is to generate a plan, validate it (with AI or human oversight), and then execute it to prevent wasted resources.
  • Reflection and Error Correction: High-performing agents use frameworks like ReAct (Reason + Act) to iterate through a cycle of Thought, Action, and Observation, allowing them to learn from outcomes.
  • Control Flows: Plans can include sequential, parallel, conditional (if), and loop-based execution for more sophisticated tasks.

(Huyen, 2025, pp. 282, 291, 293)

Effective agents often use complex patterns to decompose tasks:

Prompt Chaining: A linear sequence of LLM calls.
Routing: An initial LLM call to classify intent and direct the query.
Evaluator-Optimizer Loop: A generator LLM and an evaluator LLM work in a feedback loop.
Orchestrator-Workers: A central LLM delegates sub-tasks to specialized worker LLMs.

(Anthropic, 2024)

Best Practices for Tool Use

  • "Prompt Engineer" Your Tools: Tool definitions should be extremely clear, with examples and edge cases.
  • "Poka-yoke" (Mistake-Proof) Your Tools: Design tools to make it hard for the LLM to make common errors.
  • Start Simple and Iterate: Avoid over-engineering by starting with the simplest solution and adding complexity only when needed.

(Anthropic, 2024)

Memory in Agentic Systems

Agents require robust memory to manage information. This includes Internal Knowledge (from training), Short-Term Memory (the context window), and Long-Term Memory (external storage like a vector database).

(Huyen, 2025, p. 301)

AI Agent Protocols

The Challenge: A lack of standardized communication methods hinders agent interoperability and scalability.

The Solution: AI agent protocols are standardized frameworks that define the rules, formats, and procedures for structured communication.

(Yang et al., 2025, pp. 1, 6)

Protocol Taxonomy

1. By Object Orientation

  • Context-Oriented: Agent-to-resource interaction (e.g., MCP).
  • Inter-Agent: Agent-to-agent interaction (e.g., A2A).

2. By Application Scenario

  • General-Purpose: Broadly applicable.
  • Domain-Specific: Tailored for specific contexts (e.g., robotics).

(Yang et al., 2025, p. 7)

The Agent Communication Trilemma

Protocol design involves a trade-off between three competing goals:

Versatility

Supporting diverse message types.

Efficiency

Minimizing cost and latency.

Portability

Ensuring the protocol is easy to implement.

(Marro et al., 2024, as cited in Yang et al., 2025, p. 15)
