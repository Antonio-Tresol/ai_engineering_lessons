1. Introduction: The Art and Science of Prompting

What is Prompt Engineering?

"Prompt engineering refers to the process of crafting an instruction that gets a model to generate the desired outcome... You can think of prompt engineering as human-to-AI communication..." (Huyen, 2025, p. 211).

Anatomy of a Prompt & In-Context Learning (ICL)

  • Core parts: Task description, examples, the task itself.
  • ICL allows models to learn from examples in the prompt without updating weights.
  • Zero-Shot vs. Few-Shot: Powerful models may need fewer examples.

System vs. User Prompts & Context Efficiency

  • System Prompt: Defines persona/task (developer-provided).
  • User Prompt: Contains the immediate instruction (user-provided).
  • Critical: Use the correct model chat template to avoid performance issues.
  • "Needle in a Haystack": Models recall info best from the beginning and end of a prompt.

2. Best Practices for Crafting Effective Prompts

  1. Write Clear and Explicit Instructions
    • Define ambiguity (scoring, formats).
    • Use a persona to guide perspective.
    • Provide examples.
    • Specify the output format (e.g., JSON).
  2. Provide Sufficient Context

    Mitigates hallucinations by giving the model the necessary info to work with.

  3. Decompose Complex Tasks

    Chain smaller, simpler prompts. Improves performance, monitoring, and debugging.

  4. Give the Model "Time to Think"
    • Chain-of-Thought (CoT): Ask the model to "think step by step."
    • Self-Critique: Ask the model to evaluate its own output.
  5. Iterate, Evaluate, and Organize

    Systematically test, version prompts, and separate prompts from code for collaboration.

3. Defensive Prompt Engineering: The Threat Landscape

Why Defense is Necessary

Models can be exploited by malicious actors. Understanding the threats is the first step to building secure systems.

Main Attack Types

  • Prompt Extraction: Stealing your proprietary system prompts.
  • Jailbreaking / Prompt Injection: Tricking the model into violating its safety policies.
  • Information Extraction: Forcing the model to reveal sensitive training data or context.

Key Risks

Includes remote code execution, data leaks, misinformation, social harms, and significant brand risk.

Attack Vectors in Detail

1. Prompt Extraction & Reverse Engineering

Deducing the system prompt by analyzing outputs or tricking the model. Treat prompts as public secrets: "proprietary prompts are more of a liability than a competitive advantage" (Huyen, 2025, p. 238).

2. Jailbreaking & Prompt Injection

The most common threat. A dangerous evolution is Indirect Prompt Injection, where malicious instructions are hidden in external data (emails, websites) that the model processes.

3. Information Extraction

Getting the model to reveal sensitive data. The "Divergence Attack" (e.g., "repeat 'poem' forever") shows data can be extracted even without knowing the training set.

Defense Strategies: A Multi-Layered Approach

1. Model-Level Defense

Train models with an instruction hierarchy (e.g., System Prompt > User Prompt > Tool Output). This can neutralize many indirect injection attacks.

2. Prompt-Level Defense

Be explicit about what the model should not do. Repeat critical instructions before and after user input to reinforce them.

3. System-Level Defense

  • Isolation: Execute generated code in a sandboxed environment (virtual machine).
  • Human Approval: Require confirmation for impactful actions (e.g., database writes).
  • Guardrails: Use input/output filters to block suspicious content or PII leakage.
